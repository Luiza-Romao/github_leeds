# Meu próprio guia Markdown 📝

## O que é o GitHub Markdown?

No git, é possível usar a sua própria linguagem de marcação para gerar texto informativo. Na linhguagem Markdown, é possível utilizar diversos recursos para auxiliar na leitura.

## Recursos a serem utilizados ao escrever um arquivo com linguagem Markdown
1. O primeiro recurso é a criação de uma numbered list. Basta colocar o número e o ponto, que a visualização será criada. Em nossa bullet vamos listar pelo menos 6 recursos utilizáveis no GitHub Markdown.
2. O segundo recurso é o uso de #, que cria uma linha de título em destaque. Podem ser utilizadas até 6 hashtags em sequência para divisão em subtítulos. Exemplo:

# Título 1
## Título 2
### Título 3

---

3. Outro recurso interessante é a quebra de página, utilizada logo acima desta frase. O uso de "---" sem aspas cria uma linha que divide o conteúdo.
4. O quarto recurso é a inserção de links. [Posso criar um objeto clicável que te leve ao portal de noticias da globo, por exemplo](https://g1.globo.com/)
5. O quinto recurso é que podemos utilizar emojis 😃. Basta digitar : e inserir uma palavra que remeta ao emoji. Para achar o de lápis, por exemplo, digitamos : e logo após já escrevemos pencil para ter 📝
6. Por fim, podemos ter palavras em itálico. Basta escrever uma frase entre asteriscos. Exemplo: *Eu gosto de carne*
